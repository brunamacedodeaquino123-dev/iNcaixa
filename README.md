# Modos do Copilot

O Copilot possui diferentes modos de interação para atender necessidades específicas, desde tirar dúvidas até executar tarefas complexas de forma autônoma.

## ❓ Ask

**Objetivo:** Entender e aprender.

Use quando quiser fazer perguntas sobre código, erros, funções, arquitetura ou conceitos técnicos.

**Características:**

* Não altera o código.
* Analisa o contexto do projeto.
* Explica o funcionamento e os motivos por trás das implementações.
* Funciona como um mentor técnico.

**Quando usar:**

* Entender um erro.
* Investigar uma função.
* Aprender um conceito.
* Analisar uma stack trace.

**Resumo:** *Perguntar e compreender.*

---

## ✏️ Edit

**Objetivo:** Modificar código existente.

Você seleciona um trecho ou arquivo e descreve a alteração desejada. O Copilot aplica as mudanças diretamente.

**Casos comuns:**

* Refatoração.
* Correção de bugs.
* Melhorias de performance.
* Ajustes de lógica.
* Conversão entre linguagens.
* Adição de logs.
* Tratamento de erros.

**Resumo:** *Transformar código existente.*

---

## 🧭 Plan

**Objetivo:** Planejar antes de implementar.

Ideal para mudanças maiores ou novas funcionalidades.

**Características:**

* Divide o problema em etapas.
* Explica a estratégia.
* Valida a abordagem antes da implementação.
* Reduz riscos em alterações complexas.

**Resumo:** *Planejar antes de agir.*

---

## 🤖 Agent

**Objetivo:** Executar tarefas complexas com autonomia.

O Copilot assume uma postura mais ativa, navegando pelo projeto e realizando alterações em múltiplos arquivos.

**Características:**

* Mantém contexto entre etapas.
* Cria e modifica arquivos.
* Coordena mudanças em várias partes do projeto.
* Atua como um desenvolvedor assistente.

**Exemplo:**

> "Implemente autenticação JWT na aplicação."

O Agent identifica os arquivos necessários, cria componentes, ajusta configurações e realiza a implementação completa.

**Resumo:** *Executar tarefas amplas de forma autônoma.*

---

## 📚 Study

**Objetivo:** Aprendizado ativo.

Mais do que entregar respostas, busca desenvolver o raciocínio do usuário.

**Características:**

* Explica conceitos.
* Mostra vantagens e desvantagens das soluções.
* Faz perguntas reflexivas.
* Evolui gradualmente a complexidade do conteúdo.
* Atua como um tutor particular.

**Resumo:** *Aprender de forma guiada.*

---

## 🧠 Resumo Rápido

| Modo     | Finalidade                               |
| -------- | ---------------------------------------- |
| ❓ Ask    | Entender e tirar dúvidas                 |
| 🧭 Plan  | Planejar antes de implementar            |
| ✏️ Edit  | Alterar código existente                 |
| 🤖 Agent | Executar tarefas complexas com autonomia |
| 📚 Study | Aprender e desenvolver raciocínio        |

**Regra prática:**

* Tem uma dúvida? → **Ask**
* Quer validar uma estratégia? → **Plan**
* Precisa alterar código? → **Edit**
* Quer delegar uma implementação completa? → **Agent**
* Quer aprender profundamente? → **Study**
